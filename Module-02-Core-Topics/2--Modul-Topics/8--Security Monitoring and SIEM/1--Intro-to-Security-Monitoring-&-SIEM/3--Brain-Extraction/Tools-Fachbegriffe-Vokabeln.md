| Verwendete Tools        | - **Splunk**: Eine leistungsstarke Plattform zum Sammeln, Indexieren und Analysieren von maschinengenerierten Daten, insbesondere Sicherheits-Logs; dient als SIEM-Lösung mit Funktionen wie Echtzeit-Überwachung, Skalierbarkeit und Dashboards.<br>- **Docker**: Ein Tool zur Containerisierung, das verwendet wird, um Splunk Enterprise lokal auszuführen, ohne komplexe Installationen; ermöglicht das Pull und Run von Containern wie `splunk/splunk:latest`.<br>- **Terminal (auf macOS)**: Eine Kommandozeilen-Anwendung zum Ausführen von Befehlen wie `log stream` oder `log show --last 1h`, um System-Logs anzuzeigen.<br>- **Syslog**: Ein Standardprotokoll zum Senden von Log-Nachrichten über ein Netzwerk, verwendet von vielen Netzwerkgeräten und Linux-Systemen.<br>- **APIs (Application Programming Interfaces)**: Schnittstellen, die es SIEM-Systemen ermöglichen, Log-Daten direkt von Anwendungen oder Cloud-Diensten abzurufen.                                                                                                                                                                                                                                                                                                                       |
| Technische Fachbegriffe | - **SIEM (Security Information and Event Management)**: Ein System zur Sammlung, Zentralisierung und Analyse von Sicherheitsdaten aus verschiedenen Quellen, um Bedrohungen zu erkennen und zu reagieren.<br>- **Logs**: Digitale Aufzeichnungen von Ereignissen in Systemen, die Aktionen wie Logins, Dateizugriffe oder Netzwerkverbindungen dokumentieren.<br>- **Normalization**: Der Prozess, bei dem Logs aus unterschiedlichen Formaten in ein standardisiertes Format umgewandelt werden, um Analyse zu erleichtern.<br>- **Correlation**: Die Analyse normalisierter Ereignisse, um Muster oder Beziehungen zu identifizieren, die auf Sicherheitsvorfälle hindeuten.<br>- **Alerting**: Die Erzeugung von Benachrichtigungen, wenn verdächtige Ereignisse oder Muster erkannt werden.<br>- **Log Collection (Data Ingestion)**: Der erste Schritt in SIEM, bei dem Logs aus Quellen wie Servern, Anwendungen oder Netzwerkgeräten gesammelt werden.<br>- **Dashboard**: Eine grafische Oberfläche zur Visualisierung von Sicherheitsmetriken, Trends und Alarmen.<br>- **Event**: Ein einzelnes, diskretes Vorkommnis in einem Log, z. B. ein Benutzer-Login.<br>- **Correlation Rule**: Vordefinierte Logik, um spezifische Ereignismuster als Bedrohung zu erkennen. |
| Wichtige Vokabeln       | - **Sicherheitsüberwachung (Security Monitoring)**: Der kontinuierliche Prozess des Sammelns und Analysierens von Informationen, um Sicherheitsvorfälle zu erkennen und zu reagieren.<br>- **Wachsamkeit (Vigilance)**: Der Bedarf an ständiger Aufmerksamkeit in der Cybersicherheit, verglichen mit der Überwachung einer Stadt.<br>- **Digitale Fußspuren (Digital Footprints)**: Metapher für Logs als Spuren von Aktionen in digitalen Systemen.<br>- **Datenüberlastung (Data Overload)**: Die Herausforderung durch die enorme Menge an generierten Logs, die manuelle Analyse erschwert.<br>- **Zentrale Sichtbarkeit (Centralized Visibility)**: Ein Vorteil von SIEM, der alle Daten in einem einzigen Überblick bietet.<br>- **Bedrohungserkennung (Threat Detection)**: Die Fähigkeit, komplexe Angriffe über mehrere Systeme hinweg zu identifizieren.<br>- **Incident Response**: Der Prozess der schnellen Reaktion auf erkannte Vorfälle.<br>- **Forensik (Forensics)**: Die Analyse historischer Daten nach einem Vorfall zur Untersuchung.<br>- **Skalierbarkeit (Scalability)**: Die Fähigkeit eines Systems, große Datenmengen zu handhaben, z. B. Petabytes.                                                                                                |
