Common nmap Scan Types.
Gängige Nmap-Scan-Typen.

Transmission Control Protocol SYN Scan, sS, also known as a half-open scan.
Transmission-Control-Protocol-SYN-Scan, sS, auch bekannt als Half-Open-Scan.

It sends a SYN packet, synchronize, as if to initiate a Transmission Control Protocol connection.
Es sendet ein SYN-Paket, Synchronize, als ob es eine Transmission-Control-Protocol-Verbindung initiieren würde.

If the port is open, the target responds with SYN/ACK, synchronize/acknowledge.
Wenn der Port offen ist, antwortet das Ziel mit SYN/ACK, Synchronize/Acknowledge.

Nmap then sends an RST, reset, packet to tear down the connection before it is fully established.
Nmap sendet dann ein RST-Paket, Reset, um die Verbindung abzubrechen, bevor sie vollständig aufgebaut ist.

This makes it stealthier than a full connect scan.
Das macht es tarnender als einen vollständigen Connect-Scan.

If the port is closed, the target responds with RST.
Wenn der Port geschlossen ist, antwortet das Ziel mit RST.

If no response or an Internet Control Message Protocol unreachable error is received, the port is often marked as filtered.
Wenn keine Antwort oder ein Internet-Control-Message-Protocol-Unreachable-Fehler empfangen wird, wird der Port oft als gefiltert markiert.

This is the default scan type for privileged users in nmap.
Das ist der Standard-Scan-Typ für privilegierte Nutzer in Nmap.

Transmission Control Protocol Connect Scan, sT, completes the three-way handshake if a port is open.
Transmission-Control-Protocol-Connect-Scan, sT, vervollständigt den Three-Way-Handshake, wenn ein Port offen ist.

It is less stealthy than a SYN scan and more likely to be logged.
Er ist weniger tarnend als ein SYN-Scan und wahrscheinlicher protokolliert.

User Datagram Protocol Scan, sU, scans for open User Datagram Protocol ports.
User-Datagram-Protocol-Scan, sU, scannt nach offenen User-Datagram-Protocol-Ports.

User Datagram Protocol scanning is slower and more difficult than Transmission Control Protocol scanning because User Datagram Protocol is connectionless.
User-Datagram-Protocol-Scanning ist langsamer und schwieriger als Transmission-Control-Protocol-Scanning, weil User-Datagram-Protocol verbindungslos ist.

An open User Datagram Protocol port might not send a response.
Ein offener User-Datagram-Protocol-Port könnte keine Antwort senden.

If no response is received after multiple probes, nmap often marks it open|filtered.
Wenn nach mehreren Sondierungen keine Antwort empfangen wird, markiert Nmap es oft als open|filtered.

A closed User Datagram Protocol port should respond with an Internet Control Message Protocol port unreachable message.
Ein geschlossener User-Datagram-Protocol-Port sollte mit einer Internet-Control-Message-Protocol-Port-Unreachable-Nachricht antworten.

FIN, Null, Xmas Scans, sF, sN, sX, these are stealthier scans that send Transmission Control Protocol packets with specific flags set, FIN, no flags, or FIN, PSH, URG flags respectively.
FIN-, Null-, Xmas-Scans, sF, sN, sX, das sind tarnendere Scans, die Transmission-Control-Protocol-Pakete mit spezifischen Flags senden, FIN, keine Flags oder FIN, PSH, URG-Flags jeweils.

These rely on the behavior of some systems, mostly non-Windows, where closed ports respond with an RST, while open ports drop the packet.
Diese basieren auf dem Verhalten einiger Systeme, meist nicht-Windows, bei denen geschlossene Ports mit einem RST antworten, während offene Ports das Paket verwerfen.

Filtered ports might also drop the packet.
Gefilterte Ports könnten das Paket ebenfalls verwerfen.

Try it yourself.
Probiere es selbst aus.

Let us try a basic nmap scan against scanme.nmap.org.
Lass uns einen grundlegenden Nmap-Scan gegen scanme.nmap.org ausprobieren.

1. Open your terminal.
2. Öffne dein Terminal.

3. Run a Transmission Control Protocol SYN scan on the most common ports, this is often the default behavior or can be specified.
4. Führe einen Transmission-Control-Protocol-SYN-Scan auf den gängigsten Ports aus, das ist oft das Standardverhalten oder kann spezifiziert werden.

sudo nmap -sS scanme.nmap.org
sudo nmap -sS scanme.nmap.org

You need sudo for -sS as it crafts raw packets.
Du brauchst sudo für -sS, da es rohe Pakete erstellt.

3. Observe the output.
4. Beobachte die Ausgabe.

What ports are listed as open?
Welche Ports werden als offen aufgelistet?

What services does nmap think are running on them?
Welche Dienste denkt Nmap, dass auf ihnen laufen?

Important Note on scanme.nmap.org, while this host is provided for testing, always use it responsibly.
Wichtiger Hinweis zu scanme.nmap.org, obwohl dieser Host für Tests bereitgestellt wird, nutze ihn immer verantwortungsvoll.

Do not run aggressive or very frequent scans.
Führe keine aggressiven oder sehr häufigen Scans aus.

3. Service Enumeration and Version Detection.
4. Dienstauflistung und Versionserkennung.

Knowing a port is open is good, but knowing what service is running on that port and its version is even better.
Zu wissen, dass ein Port offen ist, ist gut, aber zu wissen, welcher Dienst auf diesem Port läuft und welche Version, ist noch besser.

Vulnerabilities are often specific to software versions.
Schwachstellen sind oft spezifisch für Software-Versionen.
