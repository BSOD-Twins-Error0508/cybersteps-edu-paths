# Packet Fishing

_Time to strain the packet soup and catch the right ones._

**Goal**

To practice writing and applying various Wireshark display filters to isolate specific types of network traffic.

**Instructions**

1. Run a wireshark capture and generate varied traffic (ping `8.8.8.8`, browse [`http://neverssl.com`](http://neverssl.com) )
2. For each scenario below, determine the correct Wireshark display filter you would use:
    - Show all traffic (both sent and received) involving the IP address `8.8.8.8`.
    - Show only HTTP traffic.
    - Show only ICMP traffic.
    - Show only traffic using TCP source or destination port 80.
    - Show only ICMP traffic originating from the IP address `8.8.8.8`.
    - Show all traffic _except_ for HTTP traffic.

**Submission**

Submit the correct display filter expression for each of the six scenarios listed.

# Paketfischen

_Zeit, die Paketsuppe zu sieben und die richtigen Pakete herauszufischen._

**Ziel**

Üben Sie das Schreiben und Anwenden verschiedener Wireshark-Anzeigefilter, um bestimmte Arten von Netzwerkverkehr zu isolieren.

**Anleitung**

1. Führen Sie eine Wireshark-Erfassung durch und generieren Sie unterschiedlichen Datenverkehr (ping `8.8.8.8`, browsen Sie [`http://neverssl.com`](http://neverssl.com) ).
2. Bestimmen Sie für jedes der folgenden Szenarien den richtigen Wireshark-Anzeigefilter, den Sie verwenden würden:
- Zeigen Sie den gesamten Datenverkehr (gesendet und empfangen) an, der die IP-Adresse `8.8.8.8` betrifft.
- Zeigen Sie nur HTTP-Datenverkehr an.
- Zeigen Sie nur ICMP-Datenverkehr an.
- Zeigen Sie nur Datenverkehr mit TCP-Quell- oder Zielport 80 an.
- Zeigen Sie nur ICMP-Datenverkehr an, der von der IP-Adresse `8.8.8.8` stammt.
- Zeigen Sie den gesamten Datenverkehr _außer_ HTTP-Datenverkehr an.

**Einreichung**

Reichen Sie den richtigen Anzeigefilterausdruck für jedes der sechs aufgeführten Szenarien ein.
