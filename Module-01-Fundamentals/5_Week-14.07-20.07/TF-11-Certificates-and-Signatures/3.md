# Signature Move

_Can't fake this autograph!_

### Goal

Use OpenSSL to create a digital signature for a file and then verify its integrity and authenticity. This exercise requires you to research the appropriate OpenSSL commands.

### Instructions

1. Create a simple text file named `my_secret_message.txt`. Write a short, unique message inside it (e.g., "The eagle flies at midnight, but only if the weather is clear.").
2. **Generate a Key Pair:**
    - Using OpenSSL, generate a new RSA private key. Save it as `signature_private_key.pem`.
    - Extract the corresponding public key from your private key. Save it as `signature_public_key.pem`.
3. **Sign the File:**
    - Using OpenSSL and your `signature_private_key.pem`, create a detached digital signature for `my_secret_message.txt`. Save the signature in a separate file named `message.sig`.
4. **Verify the Signature:**
    - First, verify the signature against the original `my_secret_message.txt` using `signature_public_key.pem`. The output should indicate success.
    - Second, to simulate tampering:
        - Create a copy of your original message file, or slightly modify `my_secret_message.txt` (e.g., add an extra word or change a character).
        - Attempt to verify `message.sig` against this _modified_ message file using `signature_public_key.pem`. The output should indicate failure.
5. What do the results of your verification attempts (successful and failed) demonstrate about digital signatures?

### Submission

1. The exact OpenSSL commands you used to:
    - Generate the private key.
    - Extract the public key.
    - Sign the file.
    - Verify the signature (both the successful and failed attempts).
2. A screenshot of your terminal showing the successful verification output.
3. A screenshot of your terminal showing the failed verification output (after modifying the message).
4. Your brief explanation (2-3 sentences) for question 5.

# Signature Move

_Diese Unterschrift kann man nicht fälschen!_

### Ziel

Verwenden Sie OpenSSL, um eine digitale Signatur für eine Datei zu erstellen und anschließend deren Integrität und Authentizität zu überprüfen. Für diese Übung müssen Sie die entsprechenden OpenSSL-Befehle recherchieren.

### Anleitung

1. Erstellen Sie eine einfache Textdatei mit dem Namen „my_secret_message.txt“. Schreiben Sie eine kurze, einzigartige Nachricht hinein (z. B. „Der Adler fliegt um Mitternacht, aber nur bei klarem Wetter.“).
2. **Schlüsselpaar generieren:**
- Generieren Sie mit OpenSSL einen neuen privaten RSA-Schlüssel. Speichern Sie ihn als „signature_private_key.pem“.
- Extrahieren Sie den entsprechenden öffentlichen Schlüssel aus Ihrem privaten Schlüssel. Speichern Sie ihn als „signature_public_key.pem“.
3. **Signieren Sie die Datei:**
- Erstellen Sie mit OpenSSL und Ihrem „signature_private_key.pem“ eine separate digitale Signatur für „my_secret_message.txt“. Speichern Sie die Signatur in einer separaten Datei mit dem Namen „message.sig“.
4. **Überprüfen Sie die Signatur:**
    - Überprüfen Sie zunächst die Signatur anhand der Originaldatei „my_secret_message.txt“ mit „signature_public_key.pem“. Die Ausgabe sollte einen Erfolg anzeigen.
- Simulieren Sie anschließend eine Manipulation:
- Erstellen Sie eine Kopie Ihrer ursprünglichen Nachrichtendatei oder ändern Sie „my_secret_message.txt“ geringfügig (z. B. durch Hinzufügen eines zusätzlichen Wortes oder Ändern eines Zeichens).
        - Versuchen Sie, „message.sig“ anhand dieser geänderten Nachrichtendatei mit „signature_public_key.pem“ zu überprüfen. Die Ausgabe sollte einen Fehlschlag anzeigen.
5. Was sagen die Ergebnisse Ihrer Überprüfungsversuche (erfolgreich und fehlgeschlagen) über digitale Signaturen aus?

### Einreichung

1. Die genauen OpenSSL-Befehle, die Sie verwendet haben, um:
- Den privaten Schlüssel zu generieren.
    - Extrahieren des öffentlichen Schlüssels.
- Signieren der Datei.
- Überprüfen der Signatur (sowohl erfolgreiche als auch fehlgeschlagene Versuche).
2. Einen Screenshot Ihres Terminals, der die erfolgreiche Überprüfungsausgabe zeigt.
3. Einen Screenshot Ihres Terminals, der die fehlgeschlagene Überprüfungsausgabe zeigt (nach dem Ändern der Nachricht).
4. Ihre kurze Erklärung (2–3 Sätze) zu Frage 5.
