What is ARP? ARP stands for Address Resolution Protocol. Its primary job is to map Layer three, Network Layer, addresses, specifically IPv4 addresses, to Layer two, Datalink Layer, addresses, typically MAC addresses, for devices sharing the same local network segment, also called broadcast domain. Was ist ARP? ARP steht für Address Resolution Protocol. Seine Hauptaufgabe besteht darin, Schicht-drei-Adressen, Netzwerkschicht-Adressen, speziell IPv4-Adressen, auf Schicht-zwei-Adressen, Datenverbindungsschicht-Adressen, typischerweise MAC-Adressen, für Geräte abzubilden, die sich dasselbe lokale Netzwerksegment teilen, auch Broadcast-Domain genannt.

Think of it as a dynamic lookup service specifically for translating IP addresses to MAC addresses within your local network. Betrachten Sie es als einen dynamischen Suchdienst speziell für die Übersetzung von IP-Adressen in MAC-Adressen innerhalb Ihres lokalen Netzwerks.

Key points about ARP: Layer. Operates at the boundary between Layer two, Datalink, and Layer three, Network. While it uses Layer two frames for transport, its purpose is to resolve Layer three addresses. Wichtige Punkte über ARP: Schicht. Es arbeitet an der Grenze zwischen Schicht zwei, Datenverbindung, und Schicht drei, Netzwerk. Während es Schicht-zwei-Frames für den Transport verwendet, besteht sein Zweck darin, Schicht-drei-Adressen aufzulösen.

Scope. ARP messages are confined to the local network segment. They are not routed across different networks by routers. Umfang. ARP-Nachrichten sind auf das lokale Netzwerksegment beschränkt. Sie werden nicht von Routern über verschiedene Netzwerke geroutet.

Stateless. ARP itself doesn't maintain a persistent connection state. It's a simple request-response protocol. Zustandslos. ARP selbst hält keinen persistenten Verbindungsstatus aufrecht. Es ist ein einfaches Anfrage-Antwort-Protokoll.

The ARP Cache. As mentioned, devices maintain an ARP cache, or table, to store learned IP-to-MAC mappings. This avoids the need to send ARP requests for every single packet destined for local devices. Der ARP-Cache. Wie erwähnt, pflegen Geräte einen ARP-Cache, oder eine Tabelle, um gelernte IP-zu-MAC-Zuordnungen zu speichern. Dies vermeidet die Notwendigkeit, ARP-Anfragen für jedes einzelne Paket zu senden, das für lokale Geräte bestimmt ist.

Dynamic Entries. Most entries are learned dynamically through the ARP request and reply process described above. These entries typically have a timeout period, often minutes, but it varies by operating system. Dynamische Einträge. Die meisten Einträge werden dynamisch durch den oben beschriebenen ARP-Anfrage- und Antwort-Prozess gelernt. Diese Einträge haben typischerweise eine Timeout-Periode, oft Minuten, aber sie variiert je nach Betriebssystem.

After the timeout, the entry is removed, and ARP resolution will need to happen again if communication is re-initiated. This helps ensure the cache doesn't hold onto stale information if a device's IP or MAC changes. Nach dem Timeout wird der Eintrag entfernt, und die ARP-Auflösung muss erneut erfolgen, wenn die Kommunikation wieder aufgenommen wird. Dies hilft sicherzustellen, dass der Cache keine veralteten Informationen behält, wenn sich die IP oder MAC eines Geräts ändert.

Static Entries. It's sometimes possible to manually add permanent entries to the ARP cache. This is less common but can be used in specific network configurations or for security purposes, though it can also cause issues if not managed carefully. Statische Einträge. Why Not Just Use IP Addresses at Layer Two? Ethernet, the most common Layer two technology for wired LANs, was designed to work with MAC addresses for local delivery. Warum nicht einfach IP-Adressen auf Schicht zwei verwenden? Ethernet, die gängigste Schicht-zwei-Technologie für verkabelte LANs, wurde entwickelt, um mit MAC-Adressen für die lokale Zustellung zu arbeiten.

Switches operate at Layer two and make forwarding decisions based on MAC addresses. They don't typically inspect Layer three IP addresses. Switches arbeiten auf Schicht zwei und treffen Weiterleitungsentscheidungen basierend auf MAC-Adressen. Sie inspizieren normalerweise keine Schicht-drei-IP-Adressen.

ARP provides the necessary bridge between the IP addressing scheme used by higher layers and the MAC addressing scheme used by Ethernet for final delivery on the local segment. ARP bietet die notwendige Brücke zwischen dem IP-Adressierungsschema, das von höheren Schichten verwendet wird, und dem MAC-Adressierungsschema, das von Ethernet für die endgültige Zustellung im lokalen Segment verwendet wird.
